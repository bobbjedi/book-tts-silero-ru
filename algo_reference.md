# Референс: алгоритм озвучки книг (Silero v5_4_ru)

## 1) Цель
Из входного художественного текста получить последовательность чанков с:
- типом: `author | line | question | exclamation`
- нормализованным текстом
- SSML (кроме `author`)

## 2) Пайплайн
1. Предобработка текста.
2. Разбор на блоки автора и прямой речи.
3. Разделение блока прямой речи на предложения.
4. Классификация предложений по типам чанков.
5. Нормализация текста (пунктуация, числа, спец-замены).
6. Генерация SSML для не-авторских чанков.

## 3) Типы чанков
- `author` — слова автора (ремарки, вводные конструкции вне прямой речи).
- `line` — обычная реплика (повествовательная).
- `question` — реплика с `?` (и комбинациями, приведенными к `?`).
- `exclamation` — реплика с `!`.

## 4) Правила сегментации
- Прямая речь выделяется по тире/контексту диалога.
- Если внутри одной реплики несколько предложений разных типов, разбивать на отдельные чанки.
- Если подряд предложения одного типа `line`, можно объединять в один чанк.
- Если у финального предложения нет знака в конце, добавить `.`.
- Слова автора между репликами выделять в отдельный `author`-чанк.

### Пример
Вход:
`- Удивительно, никогда бы не подумал. Ты действительно не жалеешь об этом? - спросил Грак. - А потом использовала ещё несколько?`

Выход:
1. `line`: `Удивительно, никогда бы не подумал.`
2. `question`: `Ты действительно не жалеешь об этом?`
3. `author`: `Спросил Грак.`
4. `question`: `А потом использовала ещё несколько?`

## 5) Интонации и SSML

### Константы (изменяемые)
- `line`: `pitch="+2%"`, `rate="+1%"`
- `exclamation`: `pitch="+4%"`, `rate="+2%"`
- `question`: `pitch="+7%"`, `rate="+2%"`

### Правила
- Для `author` SSML не добавляется, текст отдается как есть.
- Для `line | exclamation | question` контейнеры `<speak><p>...</p></speak>` обязательны.
- Для `line | exclamation | question` текст оборачивается в полный шаблон:
`<speak><p><prosody pitch="..." rate="...">...</prosody></p></speak>`
- Для `question` последнее слово перед `?` помечается `*...*`.
  - Пример: `Зачем ты это *сделала*?`

## 6) Глобальные замены
- Числа переводить в слова: `236 -> двести тридцать шесть`.
- `?!` и `!?` приводить к `?`.

## 7) Нормализация текста
- Убирать лишние пробелы.
- Начинать предложение с заглавной буквы (если возможно без потери смысла).
- Следить за финальной пунктуацией по типу чанка.

## 8) Рекомендуемый формат результата
```json
[
  {"type":"author","text":"Валенсия поерзала."},
  {"type":"line","text":"Нет.","ssml":"<speak><p><prosody pitch=\"+2%\" rate=\"+1%\">Нет.</prosody></p></speak>"},
  {"type":"author","text":"Сказала она."}
]
```

## 9) Критерии готовности
- Диалоги корректно отделены от автора.
- Смешанные реплики разбиваются на правильные типы.
- Профили `pitch/rate` применяются по типу.
- Глобальные замены выполняются стабильно.
